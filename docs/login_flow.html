Checkaroo Login Flow Documentation

This document outlines the technical flow of the authentication process in the Checkaroo application, starting from the login page.

1. Initial Access

Location: src/app/login/page.tsx

When a user navigates to /login, the server-side page component checks for an existing session cookie.

  • If a valid session cookie exists, the user is redirected to the home page (/).
  • If no session exists, the LoginPage (Client Component) is rendered within the layout.

2. User Input

Location: src/app/login/Loginform.tsx

The login form captures the username and password. It includes:

  • State management for success and error messages.
  • Keyboard listener for the "Enter" key to trigger submission.
  • A custom Button component configured to hit the authentication API.

3. Data Submission

Location: src/app/components/Button.tsx

Upon clicking "Submit", the CustomButton component performs the following:

  1. Prevents default form submission.
  2. Gathers data from the form fields (username, password).
  3. Encrypts the payload using AES encryption (via common/crypt.ts).
  4. Sends a POST request to /api/login/auth with the encrypted data and a "super-encrypted" session identifier (empty for initial login).

4. Server-Side Authentication

Location: src/app/api/login/auth/route.ts

The authentication route processes the request:

  1. Verifies request headers (referer, content-type, etc.).
  2. Decrypts the incoming data.
  3. Hashes the password using MD5.
  4. Calls validateUser(username, passwordHash) to check the User table.

5. Session Creation

Location: src/common/session.ts & src/app/api/login/auth/route.ts

If credentials are valid:

  1. Checks if the user already has an active session.
  2. If not, generates a new unique session_hash (concatenated UUIDs).
  3. Encrypts the session hash with timestamp padding.
  4. Writes the encrypted session to a cookie (7-hour expiration).
  5. Inserts a new record into the Logins table with the hash, user ID, and expiration date.
  6. Returns a success response.

6. Final Redirection

Location: src/app/login/Loginform.tsx

The handleCallBack function in the login form receives the success status:

  • Displays a success message to the user.
  • After a 1-second delay, redirects the user to the home page (/).
